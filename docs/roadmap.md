# 路线图

## Phase A：本地练习闭环

状态：基本完成。

目标是让用户能从面经文章进入问题池，完成作答、评分、重答和卡片沉淀。

当前后续只建议补稳定性和体验，不继续扩大范围：

- [ ] 修正明显 UI/交互问题。
- [ ] 补关键失败态提示。
- [ ] 继续补测试覆盖缺口。
- [ ] 清理旧日志和本地调试噪音。

## Phase B：画像和复练

状态：未开始。

建议先做只读画像报告，不急着上复杂推荐：

- [ ] 从 attempts / scores 聚合弱项。
- [ ] 生成 `data/profile/user_profile.json`。
- [ ] 展示最近得分趋势。
- [ ] 标出高频技术短板、表达短板、工程思维短板。
- [ ] 基于低分和 gap 生成复练候选题。
- [ ] Review Mode。

## Phase C：扩展和模式化

状态：未开始。

- [ ] 多信息源扩展。
- [ ] Learn Mode。
- [ ] 更完整的卡片复习体验。
- [ ] 可选数据库化。
- [ ] 可选云同步或多设备迁移。

## 暂不建议做

- 账号体系。
- 多用户协作。
- 复杂后台管理。
- 过早的推荐算法。
- 把 prompt、抓取、题库、卡片混成一个大模块。
