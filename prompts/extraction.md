# 面试问题抽取提示词 v3

## 任务

从一篇牛客面经文章中抽取“可以直接拿来练习回答”的技术面试题。

目标问题风格参考题库 card 的 `question` 字段:

- `线上慢 SQL 怎么排查?`
- `Redis 热 key 怎么处理?`
- `JDK 1.8 ConcurrentHashMap 的底层实现是什么?`
- `Java 中 == 和 equals 有什么区别?`
- `TCP 三次握手的过程是什么,为什么不是两次?`

也就是说,`question` 必须是一个独立、清晰、可回答的单句问题。

## 输入

```text
方向关键词:{{query}}
文章标题:{{title}}
文章正文:
{{text}}
```

## 输出

只返回 JSON 对象,不要 markdown 代码块、注释或散文。

如果是面经并且能抽到问题:

```json
{
  "isInterview": true,
  "questions": [
    {
      "question": "Redis 分布式锁如何实现?",
      "category": "Redis",
      "difficulty": "medium",
      "evidence": "Redis分布式锁如何实现？",
      "confidence": 0.9
    }
  ]
}
```

如果不是面经,或没有可独立练习的技术问题:

```json
{
  "isInterview": false,
  "questions": []
}
```

## 抽题规则

- 只抽技术面试题、笔试题、手撕算法题、系统设计/项目追问。
- 不抽自我介绍、反问、薪资、流程、地点、公司评价、情绪表达、求助内容。
- 每个 `question` 必须是独立问题,不能依赖“上面这个”“这个项目”等上下文。
- 不要输出带现场口语上下文的问句:
  - 不要写 `结合你的经历...`
  - 不要写 `除了这个还有什么...`
  - 不要写 `这个怎么做`
  - 不要写 `你实操过...吗`
  - 要改写成可独立练习的标准问句
- 原文是关键词时,要改写成完整问句:
  - `线程池的参数` -> `线程池有哪些核心参数,分别有什么作用?`
  - `Docker拉取镜像与创建容器的命令` -> `Docker 如何拉取镜像并创建容器?`
  - `MySQL 回表` -> `什么是 MySQL 回表,如何避免?`
- 原文是追问式表达时,改写成知识点主问句:
  - `结合你的经历，说说MySQL数据库有哪些调优方式？` -> `MySQL 数据库有哪些常见调优方式?`
  - `除了缓存击穿，还有什么问题？` -> `Redis 缓存常见问题有哪些,分别如何处理?`
- 原文已经是清晰问句时,尽量保留原句。
- 同义或重复问题合并。
- 最多抽 12 个问题。

## category

category 必须严格从下面选择一个:

- `Redis`
- `Java`
- `MySQL`
- `计网`
- `计系统`
- `Agent`

无法归类到这 6 类的问题直接丢弃,不要硬填。

## difficulty

只能使用:

- `easy`
- `medium`
- `hard`

## confidence

0 到 1。原文明确出现的问题给高分; 需要改写但依据清楚的给中高分; 推测性问题不要输出。
