# 面经标题分类提示词 v2

## 任务

给定一批牛客文章标题,逐条判断它是否应该进入“面试题抽取”流程。

你只能看标题,不能假设正文。目标是减少明显噪声,不是做最终裁决。

## isInterview=true

标题明确表示文章包含真实面试/笔试问题、面试过程或面试复盘:

- 明确出现: 面经、笔经、面试、笔试、一面、二面、三面、HR面、技术面、主管面、终面、OC、挂经
- 明确出现公司/岗位 + 面试轮次/问题,例如: 字节后端一面、美团 Java 二面、腾讯 CSIG 后台开发一面
- 标题本身就是问题列表或面试题集合,例如: 问了 Redis/MySQL/JVM/线程池、手撕算法、真题、复盘
- 口语化但明显是面试记录,例如: 已 oc、KPI 面、面试官问了、被拷打、手撕简单题

## isInterview=false

标题主要不是面试经历,或者只是在求助/讨论/招聘:

- 简历求助、求建议、求内推、求捞、求 offer 比较、薪资比较
- 招聘信息、岗位推荐、广告、内推帖
- 学习笔记、教程、八股整理、路线规划,但没有真实面试经历
- 情绪帖、行业讨论、求职形势讨论、吐槽
- 纯实习/春招/秋招求助,没有面试/笔试/轮次/问题线索

## 不确定策略

- 如果标题里有“一面/二面/三面/面试/笔试/OC/挂经/手撕/问了”等强面试信号,倾向 true。
- 如果标题很短且只有“春招 测试/测开”“实习岗有吗”“简历求建议”这类泛求职信号,判 false。
- 如果标题很长,包含多个技术问题或面试官提问,即使没有“面经”两个字,也判 true。

## 输入

```text
{{titles}}
```

每行一条,格式 `<index>. <标题>`,index 从 0 开始。

## 输出

只返回 JSON 数组,不要 markdown,不要解释文本。长度必须等于输入条数。

每项包含:

- `index`: 输入序号
- `isInterview`: boolean
- `confidence`: 0 到 1 的数字
- `reason`: 20 字以内中文短理由

```json
[
  {"index": 0, "isInterview": true, "confidence": 0.92, "reason": "含一面和技术问题"},
  {"index": 1, "isInterview": false, "confidence": 0.88, "reason": "简历求助帖"}
]
```
